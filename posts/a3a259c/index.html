Bluetooth Overview

藍芽是一個常見的無線通訊技術，作為一個由來已久的技術，其相關版本演進如下：

版本 發布年份 主要更新內容
4.0（低功耗） 2010 - 引入 Bluetooth Low Energy (BLE)，適用於物聯網和穿戴式設備。
4.1 2013 - 改善與 LTE 的干擾問題。- BLE 設備可同時作為主設備和從設備。
4.2 2014 - 提高數據傳輸速度（數據包容量增加 10 倍）。- 支持 IPv6，增強物聯網應用。
5.0 2016 - 數據傳輸速度提升至 2 Mbps。- 傳輸距離增加至 4 倍。- 廣播容量增大，適用於 Beacon 應用。
5.1 2019 - 方向尋找（AoA/AoD），提升室內精準定位能力。
5.2 2020 - LE Audio，支持 LC3 音頻編解碼器。- 增強屬性協議（EATT），提升 BLE 連接效率。
5.3 2021 - 提升 BLE 連接可靠性。- 增強功耗管理，延長電池壽命。
5.4 2023 - Periodic Advertising with Response (PAwR)，改善廣播性能。- Encrypted Advertising Data，增強安全性。
6.0 2024 - 信道探測（Channel Sounding），提供厘米級精準距離測量。- 基於決策的廣告過濾，提高掃描效率。- 增強等時適配層（ISOAL）。

藍芽基本概念

在查看藍芽相關規範時，通常會遇到以下兩個名詞：

  • BR/EDR - Basic Rate/Enhanced Data Rate
  • BLE - Bluetooth Low Energy

這兩個代表藍芽在實作上的不同規範，簡單來說，就是有不同的物理實體實作和通道管理方式。

在規範（Spec）中，也經常會看到以下兩個術語：

  • Profile - 代表一組服務或功能，用來標準化裝置間的溝通方式與狀態。例如，它描述了如何使用一個功能，包含狀態資訊等。
  • Protocol - 與網路常見的 HTTP Protocol 類似，表示裝置間如何透過封包的定義來互相傳輸資料。例如，它定義了封包結構、傳輸方式與溝通順序。

藍芽架構（Bluetooth Architecture）

藍芽的結構與一般網路結構類似，包含實體層與較高層的堆疊。通常以 HCI（Host Controller Interface） 作為軟體堆疊與硬體控制層的分界。

HCI 也對應到系統中的 HAL（Hardware Abstract Layer），負責處理驅動相關或內核（Kernel）層級的操作。

Bluetooth Core Stack

在藍芽軟體堆疊中，核心協議包含以下幾個部分：

  • GAP（Generic Access Profile）：規範如何廣播（advertising）與連接的狀態。
  • GATT/ATT（Generic Attribute Profile / Attribute Protocol）
  • SMP（Security Manager Protocol）
  • SDP（Service Discovery Protocol）
  • L2CAP（Logical Link Control and Adaptation Protocol）
  • HCI（Host Controller Interface）

在 HCI 之下，則包含 Link Manager（LM），以及負責基頻與無線電的固件控制層。

BLE 廣播與連線流程

在 BLE 中，

  • 發送廣播的設備稱為 Advertiser
  • 接收廣播的設備稱為 Scanner

當 advertising 發送 可連接（Connectable） 的廣播時，發起連線的裝置（Initiator）會發送 ACL 連線請求，然後建立連線。

此時：

  • Advertiser 接收連線後會停止 advertising event
  • 連線建立後，根據參數啟動 connection event
  • 發起連線的裝置變成 Central role，而接收連線的裝置變成 Peripheral role

Example: Advertising and Connection Event 

sequenceDiagram
    participant Advertiser
    participant Scanner
    participant Initiator
    participant Peripheral
    
    Advertiser->>Scanner: 發送 `advertising`
    Scanner->>Initiator: 掃描廣播數據
    Initiator->>Advertiser: 發送 `connection request`
    Advertiser->>Peripheral: 建立連線，變成 Peripheral
    Initiator->>Peripheral: 開始 `connection event`

藍芽在 Android AOSP 的實作

Android AOSP 中的藍芽實作位於 Fluoride 堆疊，可從 Google Source 下載：

🔗 Android Bluetooth Fluoride Source

下載後可切換到特定的 commit 或 tag，例如 Android 12.1.0_r35：

git clone https://android.googlesource.com/platform/system/bt
cd bt
git checkout android-platform-12.1.0_r35

這樣就可以查看 Android AOSP 中藍芽的具體實作！ 🚀
